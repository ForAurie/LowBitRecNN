## 标题

**有限状态激活数值系统（Finite-State Activation Numeric System, FSANS）**：一种显著缩小计算时的数值值域的神经网络前向传播方法。

有限状态激活数值系统即为：

* 基于有限离散状态重新设计数值表示与运算方式（非 IEEE754）。
* 用于优化前向传播激活函数计算部分。
* 构成一种新的数值系统。

适用于存在反函数且值域较小的激活函数，例如：sigmoid、softsign、tanh。

---

## 摘要

对于大部分神经网络，其在前向传播中大部分时间涉及到的数字的值域都很小（$1$ 至 $20$ 不等），现有神经网络数值系统（如 float16、bfloat16）仍需因个别的大值域需求不得不支持表示非常大的数字，但这些较大的状态没有被充分利用使其造成了浪费，本文提出一种方法来避免“个别的大值域需求”。

本文提出一种**有限状态激活数值系统（Finite-State Activation Numeric System, FSANS）**，其核心思想是：

1. 对于大部分神经网络，其 $weights$ 和 $biases$ 都很小，绝对值一般不超过 $20$，而且激活函数的值域也很小。

2. 将“个别的大值域需求”优化掉。

我们给出 sigmoid 类型函数的封闭更新公式，并通过**查表**实现：

* sigmoid 激活
* 激活叠加算子 (f)
* 小值域乘法

在 MNIST 全连接网络上验证该方法。实验表明，在仅使用 $13$ 位有限状态表表示的情况下，识别精度为 **92.74%**，相比原 float32 精度 **95.7%** 仅下降约 **3%**，并完全消除了运行时大值域的需求。结果表明，FSANS 能够在无需支持大值域数据类型的情况下实现神经网络前向传播，并具有潜在硬件实现价值。

## 1. 引言

目前常见的神经网络数值格式包括：

* float32
* float16 / bfloat16
* int8 量化

这些系统均遵循共同特征：

* 是比较通用的数值系统，可以解决除神经网络相关计算之外的问题。
* 计算仍在较大动态范围中进行。
* 硬件必须支持指数或大整数。

然而，对于 sigmoid、tanh 等激活函数：

$$
\operatorname{sigmoid}(x)\in[0,1]
$$

大部分时候需要的值域很小，但存储格式依旧具有大量冗余状态。这引出了一个问题：

> 神经网络真的需要“通用实数系统”吗？  
> 还是只需要一个**围绕其小值域特征设计的专用数值系统**？

本文答案是肯定的：**我们可以抛弃通用数值系统，直接围绕小值域特征定义有限状态数值系统。**

## 2. 方法

### 前言

以用于解决手写数字识别的前馈式全连接神经网络为例来讲述“方法”，该神经网络源自 [] 项目，输入层有 $28\times28=$ 个神经网络用于输入图片，中的两个隐藏层分别有 $,$ 个神经源，输出层有 $10$ 个神经元，激活函数全部使用 $\operatorname{sigmoid}$ 除输入层神经元之外，每个神经元都有独立的 $Biase$。

### 2.1 解决掉“个别的大值域需求”

对于大部分神经网络，其唯一一个出现大数字的地方是在线性变换时的求和部分，即下图 ① 中的求和部分：

![](.\1.png)

注：图中 $A_i$ 受限于激活函数值域不会很大，$W_{i,j},Biases$ 也都不大，$A_i\times W_{i,j}$ 也不大。但是经过累加后的 $B_i$ 会很大。

可以发现 $B_i$ 又要带入到激活函数，它的值域又会变的非常小，可以设计函数 $f(x,y)$ 来避免掉值域很大的 $B_i$。

构造函数 $f(x,y)$，使它满足：

$$
\operatorname{f}(\operatorname{sigmoid}(x), y)=\operatorname{sigmoid}(x+y)
$$

此时 $B_i=\dots\operatorname{f}(\operatorname{f}(\operatorname{sigmoid}(A_1\times W_{1,i}),A_2\times W_{2,i}),A_3\times{W_{3,i}})\dots$。

上式在计算过程中不会涉及到值域很大的数字，此时已解决掉了“个别的大值域需求”。

### 2.1 有限状态数值类型 SmallType

我们定义长度为 (len) 的有限状态表示：

* 1 bit 符号位
* (len-1) bit 强度位
* 映射区间 ([-V,V])

实现见 SmallType 模板：

* 支持符号
* 支持 saturating 有序比较
* 离散均匀分布

---

### 2.2 表驱动的数值算子

我们预计算并存储如下查找表：

| 表             | 功能            |
| ------------- | ------------- |
| `__SIGMOID01` | sigmoid 查表    |
| `__F`         | 组合算子 (f(a,y)) |
| `__TIMES`     | 有界乘法          |

---

### 2.3 sigmoidal 组合算子

理论基础：

[
\sigma(x+y)=\frac{1}{1+e^{-(x+y)}}
]

利用 odds 变换：

[
a=\sigma(x)
]

[
\sigma(x+y)=\frac{a}{a+(1-a)e^{-y}}
]

定义：

[
f(a,y)=\frac{a}{a+(1-a)e^{-y}}
]

我们在有限状态表中完全实现 (f)。

---

### 2.4 避免动态范围扩展

标准方式：

[
\sigma(x+y+z)
\Rightarrow x+y+z\in[-\infty,\infty]
]

我们的方式：

[
\sigma(x+y+z)=f(f(\sigma(x),y),z)
]

优势：

* 中间全程在 ([0,1])
* 无需更宽数值类型
* LUT 消除指数需求
